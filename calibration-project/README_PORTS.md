# Система калибровки корректоров - Прямое подключение к COM-портам

## Настройка для работы с реальным оборудованием

Этот проект настроен для **прямого подключения** к последовательным портам (USB/COM) через Docker.

### Требования к хост-системе

1. **Linux** (рекомендуется Ubuntu 20.04+)
2. **Docker** и **Docker Compose**
3. Права доступа к последовательным портам

### Шаг 1: Подготовка хост-системы

#### Проверка доступных портов
```bash
# Список всех USB устройств
ls -la /dev/ttyUSB*

# Список всех последовательных портов
ls -la /dev/ttyS*

# Проверка прав доступа
ls -l /dev/ttyUSB0
```

#### Добавление пользователя в группу dialout
```bash
sudo usermod -a -G dialout $USER
# Перелогиньтесь для применения изменений
```

#### Отключение менеджера модемов (если мешает)
```bash
# Ubuntu/Debian
sudo systemctl stop ModemManager
sudo systemctl disable ModemManager
```

### Шаг 2: Настройка docker-compose.yml

Файл `docker-compose.yml` уже настроен для проброса портов:

```yaml
devices:
  - "/dev/ttyUSB0:/dev/ttyUSB0"  # МИТ-8 (Канал 1, -40°C)
  - "/dev/ttyUSB1:/dev/ttyUSB1"  # МИТ-8 (Канал 2, +10°C)
  - "/dev/ttyUSB2:/dev/ttyUSB2"  # МИТ-8 (Канал 3, +60°C)
  - "/dev/ttyUSB3:/dev/ttyUSB3"  # Корректор
  - "/dev/ttyS0:/dev/ttyS0"      # Калибратор давления (опционально)
```

**Важно:** Измените映射 портов в соответствии с вашей конфигурацией оборудования.

### Шаг 3: Запуск системы

```bash
# Сборка и запуск контейнеров
cd /workspace/calibration-project
docker-compose up -d --build

# Проверка логов
docker-compose logs -f web

# Проверка доступа к портам внутри контейнера
docker exec -it calibration-system ls -la /dev/ttyUSB*
```

### Шаг 4: Проверка подключения

Откройте веб-интерфейс: http://localhost:8080

1. Перейдите в раздел "Настройки стенда"
2. Выберите COM-порты для каждого устройства:
   - **МИТ-8**: COM1 (/dev/ttyUSB0)
   - **Корректор**: COM4 (/dev/ttyUSB3)
   - **Калибратор давления**: COM5 (/dev/ttyS0) или ручной ввод
3. Нажмите "Проверить связь"

Система отправит тестовые команды на каждое устройство и отобразит результат.

### Логирование

Все операции с портами логируются:

```bash
# Просмотр логов в реальном времени
docker-compose logs -f web | grep Serial

# Логи Apache/PHP
docker exec calibration-system tail -f /var/log/apache2/error.log
docker exec calibration-system tail -f /var/www/html/logs/calibration.log
```

### Протоколы устройств

#### МИТ-8 (Термоизмеритель)
- Команда чтения температуры: `:READ1?`, `:READ2?`, `:READ3?`
- Формат ответа: ASCII строка с температурой
- Скорость: 19200 бод, 8N1

#### Корректор
- Запись точки калибровки: `CAL:TEMP <значение>`
- Чтение температуры: `MEAS:TEMP?`
- Чтение давления: `MEAS:PRES?`
- Скорость: 19200 бод, 8N1

#### Калибратор давления
- Чтение значения: `READ?`
- Настройте протокол в `SerialPortHandler.php` под вашу модель

### Отладка проблем с подключением

#### Порт не найден
```bash
# Проверка видимости порта на хосте
dmesg | grep ttyUSB

# Проверка прав
ls -l /dev/ttyUSB0
sudo chmod 666 /dev/ttyUSB0  # Временно
```

#### Порт занят другим процессом
```bash
# Найти процессы, использующие порт
sudo lsof /dev/ttyUSB0

# Остановить конфликтующие службы
sudo systemctl stop ModemManager
```

#### Проверка связи вручную из контейнера
```bash
# Вход в контейнер
docker exec -it calibration-system bash

# Установка скорости порта
stty -F /dev/ttyUSB0 19200 raw -echo

# Отправка тестовой команды
echo "?ID" > /dev/ttyUSB0

# Чтение ответа
cat /dev/ttyUSB0
```

### Конфигурация портов в приложении

Порты настраиваются в веб-интерфейсе и сохраняются в сессии:

| Устройство | Порт по умолчанию | Скорость | Четность |
|------------|------------------|----------|----------|
| МИТ-8 Канал 1 | COM1 (/dev/ttyUSB0) | 19200 | None |
| МИТ-8 Канал 2 | COM2 (/dev/ttyUSB1) | 19200 | None |
| МИТ-8 Канал 3 | COM3 (/dev/ttyUSB2) | 19200 | None |
| Корректор | COM4 (/dev/ttyUSB3) | 19200 | None |
| Давление | COM5 (/dev/ttyS0) | 9600 | None |

### Режим эмуляции (Demo Mode)

Если оборудование не подключено, система автоматически переходит в режим эмуляции:
- Генерируются случайные значения температуры/давления
- Все функции работают без реального оборудования
- Идеально для тестирования интерфейса

Для принудительного включения demo mode установите переменную окружения:
```yaml
environment:
  - CALIBRATION_DEMO_MODE=true
```

### Безопасность

- Контейнер запускается от root для доступа к устройствам
- После инициализации права передаются www-data
- Рекомендуется использовать только в доверенной сети
- Измените пароли БД в production

### Поддержка дополнительных устройств

Для добавления нового устройства:

1. Отредактируйте `src/includes/SerialPortHandler.php`
2. Добавьте методы для работы с устройством
3. Обновите mapping портов в `getLinuxPortPath()`
4. Добавьте команды в `getTestCommand()`
